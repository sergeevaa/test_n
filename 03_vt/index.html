Выявить проблемы, которые могут возникнуть при работе с продуктом и сделать все возможное для улучшения его качества

Качество ПО

[Software Quality] — совокупность характеристик ПО, которые относятся к способности продукта удовлетворять запланированным, установленным и предполагаемым потребностям.

Незачем спрашивать у тестировщика, готово ли ПО к выпуску. Принимать решение о выпуске - это работа менеджера. Работа тестировщика - снабжать информацией, достаточной для принятия такого решения.
Мы тестируем, чтобы убедиться, что от нашего ПО не откажутся из-за того, что оно не удовлетворяет основным ограничениям клиента.

Критерии качества ПО

Функциональность

способность ПО выполнять набор действий, удовлетворяющих требованиям

Надежность

способность с достаточно большой вероятностью безотказно выполнять определенные действия при заданных условиях и в течение заданного периода времени

Эффективность

соотношение между достигнутым результатом и использованными ресурсами

Эргономичность

способность минимизировать усилия пользователя по подготовке исходных данных, применению ПО и оценке полученных результатов

Модифицируемость

способность минимизировать усилия по внесению изменений для устранения ошибок и быстрота внесения новой функциональности

Мобильность

способность ПО быть перенесенным из одной среды (окружения) в другую

Практика

Задача: протестировать стул, находящийся в аудитории

    Требования к стулу:
  • Он должен cтоять на ножках
  • Он должен иметь сидушку, чтобы на нем можно было сидеть
  • Он должен иметь спинку, чтобы на нее можно было наклониться
  • Он должен быть подъемным
  • Он должен выдержать вес человека до 150 кг
  • Он должен быть пригодным для использования более 10000 раз
  • Он должен cтоять в аудитории и т.д.
